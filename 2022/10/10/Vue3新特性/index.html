一、Composition API (组合式API)

Options API存在的问题

在Vue2中使用的OptionsAPI，一旦新增或修改一个功能，就需要分别去修改data,computed中的数据以及methods中的方法，甚至修改更多的配置项，当同一组件功能较多时，代码维护十分困难

Composition API的优势
可以更加优雅的组织代码，与同一功能相关的数据与方法等能够更有序的组织在一起，重点：hook函数

setup配置

vue3中一个新的配置项，组件中的所有数据（data）、方法等均配置在setup中，并且使用return返回

<template>
  <div @click="hintMsg">{{msg}}</div>
</template>

<script>
export default {
  name: 'App',
  setup(){
    const msg = "hello world"
    function hintMsg(){
      alert(msg)
    }
    return {msg,hintMsg}
  }
}
</script>

setup的两种返回值

  1. 返回一个对象，对象中的属性与方法可以在模板中直接使用
  2. 返回一个渲染函数，则自定义渲染内容。render: h => h(APP)

setup参数

  1. props：值为对象，包括组件外部传递以及组件内部声明接收的属性
  2. context: 上下文对象
    • attrs(对象): 包含组件外部传递，但是组件内部没有接收（没在props中申明）的属性
    • slots(对象)：收到插槽的内容
    • emit(函数): 分发自定义事件的函数，相当于Vue2中的this.$emit

注意

  1. 尽量避免与Vue2配置混合使用（data(){},methods:{}等）
    • Vue2中的配置（data，methods，computed…）可以访问到setup中的属性与方法
    • 但是setup中不能访问到Vue2中的配置
    • 如果发生属性或方法重名的情况，setup中的配置优先
  2. setup不能是一个async函数，因为返回值不再是return的对象，而是promise。（当setup是一个异步函数时，需要Suspense配合异步组件进行使用）
  3. setup执行的时机：在beforeCreate钩子之前执行一次，所以在setup中使用this得到的是undefined

ref与reactive

ref函数

作用：定义一个响应式数据

语法：

// 引入ref函数
import {ref} from "vue"
// 定义一个响应式数据
let msg = ref(“hello”)
// 修改数据
msg.value = "hi"
  1. ref函数创建了一个包含响应式数据的引用对象（refimpl）
  2. js中操作数据时要通过 xxx.value
  3. 模板中读取数据时不需要.value，直接使用即可。

注意：

  1. 接收的数据类型可以是基本类型和引用类型（对象类型）
  2. 基本数据类型的响应式依然是通过Vue2中的数据劫持实现的，通过Object.defineProperty()为数据添加get set方法
  3. 对象数据类型的响应式：内部使用了Vue3中的一个新函数——reactive函数

reactive函数

作用：定义一个对象类型的响应式数据，基本类型不能使用reactive

reactive函数接收一个数组或对象，并返回一个代理对象（Proxy的实例对象）

内部基于ES6的Proxy实现，通过代理对象操作源数据的内部数据

示例：

<template>
  <div>
    <span>person1:{{person1.name}}</span>
    <span>person1:{{person1.age}}</span>
    <br/>
    <span>person2:{{person2.name}}</span>
    <span>person2:{{person2.age}}</span>
  </div>
  <button @click="change">修改信息</button>
</template>

<script>
import {reactive, ref} from 'vue'

export default {
  name: "TestApp",
  setup(){
    let person1 = ref({
      name:"张三",
      age: 18
    })
    let person2 = reactive({
      name:"张三",
      age: 18
    })
    function change() {
      person1.value.name = "王二"
      person1.value.age = 20
      person2.name = "李四"
      person2.age = 30
    }
    return {person1,person2,change}
  }
}
</script>

vue3的响应式原理

  1. 通过Proxy（代理）：拦截对象中任意属性的变化，包括值的读写，属性的添加，属性的删除等。
  2. 通过Reflect（反射）：对被代理的对象属性进行操作
 const person = {
    name: "小明",
    age: 23
}
// 模拟vue3中实现响应式
const newPerson = new Proxy(person,{
    get(target, propName) {
        console.log(`${propName}被获取了`)
        return Reflect.get(target,propName)
        // return target[propName]
    },
    set(target, propName, value) {
        console.log(`${propName}被修改了`)
        return Reflect.set(target,propName,value)
        // target[propName] = value
    },
    deleteProperty(target, propName) {
        console.log(`${propName}被删除了`)
        return Reflect.deleteProperty(target,propName)
        // delete target[propName]
    }
})

computed

使用组合式api的方式，此时的computed不是一个配置，而是一个函数

<template>
  <p>firstName:<input type="text" v-model="person.firstName"></p>
  <p>lastName:<input type="text" v-model="person.lastName"></p>
  <p>fullName {{person.fullName}}</p>
  <p>fullName:<input type="text" v-model="person.fullName"></p>
</template>

<script>
import {computed, reactive} from 'vue'

export default {
  name: "TestApp",
  setup(){
    const person = reactive({
      firstName:"张",
      lastName:"三四"
    })
    // let fullName = computed(()=>{
    //   return person.firstName + "-" + person.lastName
    // })

    person.fullName = computed({
      get(){
        return `${person.firstName}-${person.lastName}`
      },
      set(value){
        const nameList = value.split("-")
        person.firstName = nameList[0]
        person.lastName = nameList[1]
      }
    })
    return {
      person
    }
  }
}
</script>

watch与watchEffect

watch

注意：

  1. 监视reactive定义的响应式数据时，oldValue无法正确获取，且强制开启深度监视，deep配置失效
  2. 监视reactive定义的响应式数据中的某个属性时，deep配置有效
<template>
  <h3>
    <span>计数：{{num}}</span>
    <span>薪资：{{pay}}</span>
  </h3>
  <button @click="num++">点击计数+1</button>
  <button @click="pay+=1000">点击薪资+1000</button>
  <br>
  <br>
  <div>
    <h3>姓名：{{person.name}}</h3>
    <h3>年龄：{{person.age}}</h3>
  </div>
  <button @click="person.name += '!'">点击修改姓名</button>
  <button @click="person.age += 1">点击修改年龄</button>

</template>

<script>
import {reactive, ref, watch} from 'vue'

export default {
  name: "TestApp",
  setup(){
    let num = ref(0)
    let pay = ref(3000)
    const person = reactive({
      name: "张三",
      age: 18
    })

    // 方式一:监视由ref定义的一个响应式数据
    // watch(num,(newVal,oldVal) => {
    //   console.log("num发生了变化",newVal,oldVal)
    // })

    // 方式一:监视由ref定义的多个响应式数据
    // watch([num,pay],(newVal,oldVal) => {
    //   console.log("num发生了变化",newVal,oldVal)
    // },{immediate:true})

    /*
    * 方式三: 监视由reactive定义的对象的全部属性
    * 无法获取正确的oldValue
    * 默认强制开启深度监听
    * */
    // watch(person,(newVal,oldVal) => {
    //   console.log("num发生了变化",newVal,oldVal)
    // })

    /*
    * 方式四:监视由reactive定义的对象的某一个属性
    * */
    // watch(()=>person.name,(newVal,oldVal)=>{
    //   console.log("person.name被修改了",newVal,oldVal)
    // })

    /*
    * 方式五:监视由reactive定义的对象的多个属性
    * */
    watch([()=>person.name,()=>person.age],(newVal,oldVal)=>{
      console.log("person.name和person.age被修改了",newVal,oldVal)
    })
    
    return {num,pay,person}
  }
}
</script>

watchEffect

watch：指明需要监视的属性，以及属性变动后的回调

watchEffect：不用指明监视的属性，监视的回调中用到的属性变化时，则执行回调函数

watchEffect默认开启immediate

watchEffect与计算属性的区别：

  1. computed依赖于函数返回值，必须有返回值
  2. watchEffect依赖于函数过程，不用返回值
watchEffect(()=>{
  let testVal = num.value * 10
  // 当watchEffect中的依赖属性num发生修改时，便会重新执行回调函数
  console.log("watchEffect提示：num被修改了",testVal)
})

生命周期

vue3中依然可以通过配置项的形式继续使用Vue2中的生命周期钩子，但是有两个被改名了

  • beforeDestory ===> beforeUnmount
  • destoryed ===> unmounted

去掉了销毁的概念，改为卸载

Vue3提供了Composition API形式的生命周期钩子，与Vue2中的钩子对应如下

  • beforeCreate ===> setup()
  • created ===> setup()
  • beforeMount ===> onBeforeMount
  • mounted ===> onMounted
  • beforeUpdate ===> onBeforeUpdate
  • updated ===> onUpdated
  • beforeUnmount ===> onBeforeUnmount
  • unmounted ===> onUnmounted

Vue3并没有提供beforeCreate和created这两个生命周期的组合式api，均使用setup进行了替代

自定义hook函数

hook本质是一个函数，把setup函数中使用的组合式api进行了封装

类似vue2中的mixin，能够实现代码复用

自定义hook，一般使用useXXX.js命名hook文件

import {onBeforeUnmount, onMounted, reactive} from 'vue'
export default function (){
    const point = reactive({
        x: 0,
        y: 0
    })
    function savePoint(event){
        point.x = event.pageX
        point.y = event.pageY
    }
    onMounted(()=>{
        window.addEventListener("click",savePoint)
    })

    onBeforeUnmount(()=>{
        window.removeEventListener("click",savePoint)
    })
    return point
}

在组件中使用自定义的hook

<template>
  <p>获取当前鼠标坐标点:[x:{{point.x}},y:{{point.y}}]</p>
</template>

<script>

import usePoint from '@/hooks/usePoint'

export default {
  name: "TestApp",
  setup(){
    const point = usePoint()
    return {point}
  }
}
</script>

toRef

作用：创建一个ref对象，其value值指向另一个对象中的某个属性

应用：希望将响应式对象中的属性单独返回给外部进行使用的时候

使用：

<template>
  <p>姓名：{{name}}</p>
  <p>年龄：{{age}}</p>
  <p>性别：{{sex}}</p>
  <p>{{person}}</p>
  <button @click="name+='!'">修改姓名</button>
  <button @click="age++">修改年龄</button>
  <button @click="sex+='~'">修改性别</button>
</template>

<script>

import {reactive, toRef} from 'vue'

export default {
  name: "TestApp",
  setup(){
    const person = reactive({
      name: "张三",
      age:19,
      sex:"男"
    })
    return {
      person,
      name:toRef(person,"name"),
      age:toRef(person,"age"),
      sex:toRef(person,"sex")
    }
  }
}
</script>

拓展：toRef只能创建一个ref对象，toRefs可以返回多个ref对象

示例：

<template>
  <p>姓名：{{name}}</p>
  <p>年龄：{{age}}</p>
  <p>性别：{{sex}}</p>
  <p>{{person}}</p>
  <button @click="name+='!'">修改姓名</button>
  <button @click="age++">修改年龄</button>
  <button @click="sex+='~'">修改性别</button>
</template>

<script>

import {reactive, toRefs} from 'vue'

export default {
  name: "TestApp",
  setup(){
    const person = reactive({
      name: "张三",
      age:19,
      sex:"男"
    })
    return {
      person,
      ...toRefs(person)
    }
  }
}
</script>

customRef

自定义Ref，并对其依赖跟踪和更新触发进行显示控制

使用自定义Ref完成一个防抖示例

可以定义一个hook

import {customRef} from 'vue'
export default function(value,delay=500){
    let timer = null
    return customRef((track, trigger)=>({
        get(){
            track() // 跟踪value
            return value
        },
        set(newVal){
            clearTimeout(timer)
            value = newVal
            timer = setTimeout(()=>{
                trigger() // 触发模板解析
            },delay)
        }
    }))
}

然后在组件中使用

<template>
  <input type="text" v-model="inputVal">
  <p>{{inputVal}}</p>
</template>

<script>
import debounceRef from "@/hooks/useDebounceRef"
export default {
  name: "TestApp",
  setup(){
    let inputVal = debounceRef("hello")
    return {
      inputVal
    }
  }
}
</script>

provide与inject

作用：实现祖孙组件之间通信

使用：父组件使用provide提供数据，后代组件使用inject拿到数据

父级组件发送数据

setup(){
  const data = reactive({
    name:"张三",
    age: 18
  })
  provide("personData",data)
  return {
    ...toRefs(data)
  }
}

后代组件接收数据

setup(){
  const person = inject("personData")
  return{
    person
  }
}

父子组件之间同样可以使用provide与inject

其他Composition API

shallowReactive 与shallowRef

  • shallowReactive：只处理对象最外层的响应式，不处理更深层次的属性
  • shallowRef：只处理基本数据类型的相应书，不处理对象类型的响应式

readonly与shallowReadonly

  • readonly：让一个响应式数据变为深层次的只读
  • shallowReadonly：让一个响应式数据变为浅层次的只读

toRaw 与 markRaw

  • toRaw：返回一个响应式数据的原始对象，对普通对象的操作不会引起页面的更新
  • markRaw：给一个不希望成为响应式对象的原始对象打标记，被标记后的原始对象不会被proxy变为响应式对象，渲染不可变的大型数据源时，可提升效率

响应式数据的判断

  • isRef：检查一个值是否为ref对象

  • isReactive：检查一个对象是否是由reactive创建的响应式代理

  • isReadonly：检查一个对象是否是由readonly创建的只读代理

  • isProxy：检查一个对象是否是由reactive或readonly方法创建的代理

二、新的内置组件

Fragment

在Vue2中，组件必须有一个根标签，template中不允许嵌套多个元素

Vue3中，组件可以没有根标签，内部会通过Fragment虚拟元素包裹多个标签，减少标签层级，较少内存占用

Teleport

Teleport能够将我们的组件的html结构移动到指定的位置

使用Teleport实现一个弹窗

<template>
  <button @click="show = true">点击弹窗</button>
  <teleport to="body">
      <!--使用穿梭框将结构移动到body标签内-->
    <div v-if="show" class="mask">
      <div class="dialog">
        <p>我是一个弹窗</p>
        <hr>
        <button @click="show = false">关闭弹窗</button>
      </div>
    </div>
  </teleport>
</template>

<script>/**
 * @author zhaojiuyi
 * @date 2022/10/10-16:31
 * @version 1.0.0
 */
import {ref} from 'vue'

export default {
  // eslint-disable-next-line vue/multi-word-component-names
  name: 'Dialog',
  setup(){
    let show = ref(false)
    return {
      show
    }
  }
}
</script>
<style>
.mask{
  position: absolute;
  top: 0;left: 0;right: 0;bottom: 0;
  background: rgba(0,0,0,.5);
  display: grid;
  place-items: center;
}
.dialog{
  width: 200px;
  aspect-ratio: 3/2;
  background: aqua;
  position: relative;
}
</style>

Suspense

等待异步组件时渲染一些额外内容，提升用户体验

<template>
 <div class="box">
   <h1>父组件</h1>
   <suspense>
     <!--默认插槽，放入异步组件-->
     <template v-slot:default>
       <AsyncTest></AsyncTest>
     </template>
	<!--异步组件未加载完成时，使用fallback插槽里面的内容-->
     <template v-slot:fallback>
       <h1>加载中...</h1>
     </template>
   </suspense>
 </div>
</template>

<script>
import {defineAsyncComponent} from 'vue'
/*定义一个异步组件*/
const AsyncTest = defineAsyncComponent(() => import("./components/AsyncTest"))
export default {
  name: 'App',
  components: {AsyncTest},
}
</script>

三、其他改变

全局api的转移

Vue2全局API Vue3实例API
Vue.config.xxx app.config.xxx
Vue.config.productionTip 移除
Vue.component app.component
Vue.directive app.directive
Vue.mixin app.mixin
Vue.use app.use
Vue.prototype app.config.globalProperties

data选项应始终被声明为一个函数

过渡类名的修改

  • Vue2
.v-enter
.v-enter-to
.v-enter-active
.v-leave
.v-leave-to
.v-leave-active
  • vue3
.v-enter-from
.v-enter-to
.v-enter-active
.v-leave-from
.v-leave-to
.v-leave-active

移除keyCode支持作为v-on的修饰符,同时也不支持config.keyCodes

移除v-on.native修饰符，如果父组件在子组件标签中绑定了自定义事件，则应该在子组件中使用emits配置项接收，否则认为是一个原生事件

过滤器（filter）被移除
