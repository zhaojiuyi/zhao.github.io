$nextTick

当我们更新数据时，为了并不会立刻解析模板，
这是为了防止当我们同时修改多个数据时促使vue频繁解析模板与更新dom。
需要等到当前语句执行完成后，vue才会帮我们解析模板，
$nextTick 接收一个回调函数，该回调函数在dom更新完成后进行调用，
防止我们在vue未完成dom更新的时候对新的dom元素做一些操作

// 伪代码，假设有一个Test输入框组件，通过isEdit控制显示与否,使用refs="test"标识了组件
hadnleEdit(){
    this.isEdit = true  //将组件进行显示
    this.$refs.test.focus()  //输入框聚焦
}
/*
* 此时会出现一个问题，当this.isEdit = true 执行时，vue并不会立即解析模板，显示该输入框，而是要等到当前任务执行完成之后。
* 所以只有执行完成 this.$refs.test.focus() 后，输入框才会被渲染在页面上，由于输入框并没有出现在页面上，所以该代码就无效
* 使用$nextTick钩子，可以将聚焦操作放到组件渲染完成后执行
*/ 
hadnleEdit(){
    this.isEdit = true  //将组件进行显示
    this.$nextTick(function(){
        this.$refs.test.focus()  //输入框聚焦
    })
}

transition过渡

进入时的类名
v-enter
v-enter-active
v-enter-to
从v-enter(起点)到v-enter-to(终点)
整个过程为v-enter-action

离开时的类名

v-leave-active
v-leave-to

transition标签内只能嵌套一个元素
如果想嵌套多个元素，则使用transition-group

插件开发

export default {
  install(Vue){
    /*自定义指令*/
    Vue.directive("focus",{
      inserted(dom){
        // 自动获取焦点
        dom.focus()
      }
    })

    /*自定义过滤器*/
    Vue.filter("strTailor",(value,len=10) => {
      // 字符串长度限制
      if (String(value).length <= len) return value;
      return String(value).slice(0,len) + '...'
    })
  }
}

前端跨域

开启代理服务器（一）

在vue.config.js中配置

module.export = {
    devServer: {
        proxy: "http://localhost:8443" // 后端服务器地址
    }
}

当配置代理后，vue-cli会在本地开启一个服务器，通过该服务器转发请求，并返回响应数据，此时，前端不在直接向后端发送请求，而是向代理服务器发送请求

例如当前项目启动地址为 http://localhost:8080，那么向后端发送请求的地址就应该为http://localhost:8080 + 接口名

需要注意的是，代理服务器的地址与端口与项目启动的地址和端口相同，所以不会产生跨域问题

优点： 配置简单，请求资源时直接发给前端项目地址即可

缺点： 不能配置多个代理，不能灵活控制请求是否通过代理（如果请求的资源本地存在，则直接返回本地数据）

开启代理服务器（二）

devServer: {
    proxy: {
        '/api1': { // 匹配所有以 '/api1' 开头的请求路径，不论本地是否存在资源，强制转发
            target: "http://localhost:8443",
            changeOrigin: true, // 将服务器收到的请求头中的host改为与服务器一样，可有效防止服务器对来源做限制，默认为true
            pathRewrite: {"^/api1": ""} // 使用正则将路径前缀进行替换
        },
        '/api2': { 
            target: "http://localhost:8686",
            changeOrigin: true, 
            pathRewrite: {"^/api2": ""} 
        }
    }
}

插槽

默认插槽

子组件定义插槽

<template>
	<div>
        <slot>当父组件没有往插槽写入内容时，则展示这一段文字</slot>
    </div>
</template>

父组件使用插槽

<template>
	<child>
    	<span>子组件标签内的内容会放到子组件的插槽内</span>
    </child>
</template>

适用于一个插槽的情况，如果子组件具有多个插槽，便会导致子组件标签内的内容被渲染多次

具名插槽

子组件定义插槽

<template>
	<div>
        <slot name="test">当父组件没有往插槽写入内容时，则展示这一段文字</slot>
    </div>
</template>

父组件使用插槽

方式一（标签内只有一个元素时）：

<template>
	<child>
    	<span slot="test">子组件标签内的内容会放到子组件的插槽内</span>
    </child>
</template>

方式二（标签内有多个元素时可使用template）：

<template>
	<child>
    	<template slot="test">
        	<span>子组件标签内的内容会放到子组件的插槽内</span>
            <h1>这是一段标题</h1>
        </template>
    </child>
</template>

方式三（使用template时的另外一种写法）：

<template>
	<child>
    	<template v-slot:test>
        	<span>子组件标签内的内容会放到子组件的插槽内</span>
            <h1>这是一段标题</h1>
        </template>
    </child>
</template>

作用域插槽

如果希望在父组件中使用子组件的数据生成插槽的内容，可以使用作用域插槽

子组件定义

<template>
	<div>
        <slot :list="list"></slot>
    </div>
</template>
<script>
export default{
    data(){
        return {
            list: ["吃饭","睡觉","打豆豆"]
        }
    }
}
</script>

父组件使用

<template>
	<div>
        <child>
            <!-- 作用域插槽必须使用template来接收数据，scope是默认属性，值可以随意定义，用来接收子组件传递的参数 -->
            <template scope="data">
                <!-- 此时的data是一个对象，data = {"list": ["吃饭","睡觉","打豆豆"]} -->
                <ul>
                    <li v-for="item in data.list">{{item}}</li>
    			</ul>
			</template>
    	</child>
		<child>
            <!-- 作用域插槽必须使用template来接收数据，scope是默认属性，值可以随意定义，用来接收子组件传递的参数 -->
            <template scope="data">
                <!-- 此时的data是一个对象，data = {"list": ["吃饭","睡觉","打豆豆"]} -->
                    <h1 v-for="item in data.list">{{item}}</h1>
			</template>
    	</child>
    </div>
</template>

<router-link to="`/test?username=${username}`"></router-link>

<router-link to="{path:'/test',query:{username:'张三',age:18}}"></router-link>

this.$route.query

params

传递方式一，拼接url

// 路由配置
{
    path: "/test/:name/:age",
    component:Test,
}
<router-link to="/test/张三/18"></router-link>

传递方式一，使用params配置

如果要使用parmas配置传递参数，必须使用name进行跳转

<router-link to="{name:'test',params:{username:'张三',age:18}}"></router-link>

props

路由配置

{
    name:"test",
    path: "/test",
    component: Test,
    // props第一种写法，该对象中的所有key-value都会通过props的形式传递给Test组件，可直接在组件中使用props配置接收参数
    // props: {a:1,b:'hello'},
        
    // props的第二种写法，值为布尔值，若布尔值为真，就会把该路由组件接收到的所有params参数通过props形式传递给Test组件
    // props: true
   	
    // props的第三种写法，值为方法，该方法接收一个参数，参数为$route，返回一个对象，将对象的key-value通过props传递给Test组件
    props($route){
        // return {a:$route.params.a,b:$route.params.b}
        return {a:$route.query.a,b:$route.query.b}
    }
}

keep-alive

用于缓存组件，当路由切换时，不展示的组件将会被销毁，当使用 keep-alive 标签包裹时，组件则不会执行销毁。

<keep-alive>
    <!-- 默认缓存当前router-view渲染的所有组件 -->
	<router-view></router-view>
</keep-alive>
<keep-alive include="Test">
    <!-- 只缓存Test组件 include配置为组件名，也就是组件配置里面的name属性 -->
	<router-view></router-view>
</keep-alive>

<keep-alive :include="['Test1','Test2']">
    <!-- 缓存多个组件,使用数组写法 -->
	<router-view></router-view>
</keep-alive>

Vue路由器的两种工作模式

hash

hash值（对于一个url来说，#后面的内容就是hash值）不会包含在http请求中，例如

http://localhost:8080/#/user

当我们刷新页面时，浏览器不会向服务器发送#后的内容，资源能正常返回

但是如果没有hash，地址后面的所有参数路径都会发送给服务器，可能导致资源不存在

例如

http://localhost:8080/user

其中的user只是前端使用的路由地址，实际的资源地址应该是http://localhost:8080/,但是浏览器刷新后会向localhost:8080下的/user这个路径去寻找资源

对于hash来说：

  1. 地址中永远带着#,不美观
  2. 若地址通过第三方app进行分享时，可能会被标记为不合法
  3. 兼容性较好。

history

history没有hash的#，所以当部署上线后，刷新页面时会将路径的所有内容发送到服务器，导致404问题。

  1. 地址干净，美观
  2. 应用部署上线时稍微麻烦，需要后端人员的支持
  3. 兼容性相比hash较差

解决方案：nodejs中间件：connect-history-api-fallback，nginx………
