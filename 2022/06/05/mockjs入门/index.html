mockjs基本使用

mockjs简介

官方文档 http://mockjs.com/

mockjs两大特性

数据类型丰富
可以生成随机的文本、数字、布尔值、日期、邮箱、链接、图片、颜色等。

拦截 Ajax 请求
不需要修改既有代码，即可以拦截 Ajax 请求，并返回模拟的响应数据。

使用场景

在实际的开发过程中，有时候后端接口还未开发完成，而前端又不想写静态页面，为了不影响前端开发进度。我们就可以使用mockjs进行后端接口和数据的模拟，mockjs可以拦截前端的ajax请求并返回生成的随机数据，以达到模拟后端接口的效果。

当后端接口开发完成时，我们只需要取消mockjs的拦截作用，便能访问到真实的后端接口，并拿到后端真实的数据。

创建项目

首先我们电脑上需要安装nodejs，为我们提供本地运行环境

现在开始创建项目

WebStrom创建

注：本文使用WebStrom作为开发工具，没有WebStrom的同学请跳过本小结，下一小节介绍其他方式创建项目。

文件 → 新建 → 项目，选择nodejs，设置项目文件夹（文件夹为空），配置解释器和包管理器，这些只要安装了且正确配置了nodejs，WebStrom都会自动配置。下一步创建。

在打开的项目中我们能看到一个package.json文件,在里面添加如下依赖

"dependencies": {
    "mockjs": "^1.1.0",
    "require": "^2.4.20"
 }

然后打开终端，运行npm install导入依赖，如下图表示成功，接下来会在项目根目录多出一个node_modules文件夹。此时我们的运行环境便搭建好了。

其他方式创建

首先我们在目录下创建一个空文件夹，然后进入文件夹，在地址栏输入cmd，回车，即可使用命令行进行项目的创建。

在打开的cmd命令窗口中，我们输入npm init，关于项目一些配置暂时先不用管，这里直接一路回车

此时回到刚刚创建的文件夹下，刚刚的cmd窗口先不要关，可以看到生成了一个package.json文件，使用编辑器打开（记事本，sublime皆可），在里面添加如下内容

"dependencies": {
    "mockjs": "^1.1.0",
    "require": "^2.4.20"
  }

保存后继续回到cmd窗口，运行npm install导入相关依赖，完成后可以看到项目根目录下创建了node_modules文件，现在环境就搭建完成了，开始使用mockjs。

hello mockjs

在项目下创建 mock.js，编写如下代码

const Mock = require("mockjs")

let data = Mock.mock({
    "data": "hello mockjs!"
})
console.log(data)

运行可以看到如下结果

使用cmd进入项目根目录，通过node mock.js命令我们也可以运行mock.js文件，下同

当然，这样的数据肯定不能满足我们的需求

数据模板

常用api

Mock.mock(url, type, data)

参数名 描述 例子
url 非必填，需要拦截的ajax请求地址 /login, /mock/getdata
type 非必填，请求类型 post,get
data 非必填，返回的响应数据 对象或字符串

数据生成

使用require导入mockjs

const Mock = require("mockjs")

自定义字符串

// key|规则：value  
let data1 = Mock.mock({  //生成s个”测试文字“字符串
    "data|3":"测试文字。"
})
console.log(data1)
/**
* { data: '测试文字。测试文字。测试文字。' }
*/

let data2 = Mock.mock({"String|1-4":"测试文字"})   // 随机生成1-4个”测试文字“字符串

console.log(data2)
/**
* { data: '测试文字。测试文字。' }
*/

随机字符串

let data = Mock.mock({
    "data1":"@cword", //单个字符
    "data2":"@cword(4)", //指定长度字符串
    "data23":"@cword(4,8)" //指定长度范围字符串
})
console.log(data)
/**
* { data1: '干', data2: '派管据制', data23: '每调今要行' }
*/

随机省份

let data = Mock.mock({  //生成随机省份
    "province":"@province"
})
console.log(data) //{ province: '新疆维吾尔自治区' }


// 这个示例只返回对应的value，而不是对象，可根据不同需求进行使用，其他模板也可以这样使用
// 下面就不再进行演示了
let province = Mock.mock("@province") 
console.log(province) //贵州省

标题和句子

let data = Mock.mock({
    "data1":"@ctitle", //标题
    "data2":"@csentence", //句子
    "data3":"@ctitle(10)", //指定长度标题
    "data4":"@csentence(50)", //指定长度句子
})
// 除了指定长度，指定范围同样可用，语法与前面一致

段落

let data = Mock.mock({
    "data1":"@cparagraph", //段落
})

图片地址

let data = Mock.mock({
    "image": "@image('1000x1000', '#ffffff', '#df7bad', 'hello-mockjs')"
    //可选参数 尺寸、背景色、文字颜色、文字内容
})

随机时间

let data = Mock.mock({
    "time1": '@date()', // 年月日
    "time2": '@date(yyyy-MM-dd hh:mm:ss)'  //指定格式
})

个人信息

let data = Mock.mock({
    "name":"@cname",  //姓名
    "age":"@integer(18,60)", // 指定年龄范围，也可以不指定
    "city":"@city(true)", //@ city只会生成城市，加上true则加上省，市
    "idCard":"@id",  //身份证
})

对象列表

let data = Mock.mock({
    "data|5":[{
        "key|+1": 1
    }]
})
console.log(data)
/**
* {
* 	data: [ { key: 1 }, { key: 2 }, { key: 3 }, { key: 4 }, { key: 5 } ]
* }
*/

在Vue项目中使用

打开自己的Vue项目，在src目录下创建mock文件夹，然后在文件夹下创建mock.js文件，用于编写模拟数据的代码。

添加如下代码

import Mock from 'mockjs'
 //mock
let data = Mock.mock({
  id: "@id()", // 得到随机的id,对象
  username: "@cname()", // 随机生成中文名字
  date: "@date()", // 随机生成日期
  avatar: "@image('200x100', '#00405d', '#FFF', 'Mock.js')",
  // => "http://dummyimage.com/200x100/00405d/FFF&text=Mock.js"
  // description: "@paragraph()",// 描述
  ip: "@ip()",// IP地址
  email: "@email()" // email

})
Mock.mock("http://localhost:8080/login", 'post', {  //拦截请求，返回模拟的响应数据
  "token": "12asdf41asfasf2123asf4312safasdff34",
  "data" : data
})

其中，http://localhost:8080/login是我们要拦截的接口地址，post表示需要拦截的请求方式 。

然后在项目的main.js文件导入mock.js文件

import '@/mock/index.js/'

然后进行测试，在页面中调用接口

axios.post("http://localhost:8080/login").then(response => {
   console.log(response)
})

在浏览器打开调试工具

成功！！！
